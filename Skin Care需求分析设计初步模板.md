# Skin-care 护肤微信小程序需求分析设计文档

## 1. 项目概述

### 1.1 项目名称

Skin-care 微信小程序

### 1.2 项目背景

为谷雨品牌的产品体验官及广大护肤品用户打造一个集皮肤检测、产品认知、个性化护肤方案及体验反馈于一体的智能服务平台。旨在通过科技赋能，提升用户体验，深化品牌专业形象，并收集宝贵的一线用户数据。

### 1.3 项目目标

- **短期目标**：为用户提供精准的皮肤分析、个性化的产品推荐与护肤指导，提升用户粘性与品牌满意度。
- **长期目标**：构建完整的用户护肤数据画像，为产品研发、市场策略提供数据支持，并建立一个高活跃度的品牌用户社群。

### 1.4 目标用户

- **核心用户**：谷雨品牌的产品体验官、对成分有研究需求的护肤品爱好者。
- **次要用户**：寻求科学护肤指导的普通消费者。

---

## 2. 用户角色与核心场景

### 2.1 用户角色：资深体验官「小敏」

- **特征**：25-35岁女性，成分党，乐于分享，对护肤有强烈的研究欲。
- **目标**：深度了解产品机理，科学记录体验效果，与品牌和其他体验官交流。
- **核心场景**：
  1.  收到新品试用任务，通过小程序查询成分表和使用指南。
  2.  定期进行皮肤检测，记录皮肤状态变化趋势。
  3.  在护肤日记中记录使用感受，并提交格式化的体验报告。
  4.  在社群中分享心得，查看自己的体验官积分与排名。

### 2.2 用户角色：护肤新手「小雨」

- **特征**：18-25岁女性，护肤知识薄弱，皮肤存在一些问题（如痘痘、干燥），寻求简单有效的解决方案。
- **目标**：了解自己的皮肤，获得一套清晰、可信的护肤流程和产品推荐。
- **核心场景**：
  1.  首次进入小程序，通过AI皮肤检测全面了解自己的肤质和问题。
  2.  根据系统生成的护肤方案，购买并学习使用推荐的产品。
  3.  每日记录简单的护肤流程，接收小程序根据天气、生理期等场景推送的护肤贴士。
  4.  通过成长体系，激励自己持续护肤。

## 3. 功能需求详述

### 3.1 模块一：用户与档案中心

- **FR1.1 微信一键登录**：快速创建账户，绑定手机号用于后续服务。
- **FR1.2 个性化皮肤档案创建**：
  - **AI智能检测**：通过手机摄像头拍照，分析肤质、肤色、皱纹、痘痘、黑眼圈、毛孔等多项指标。
  - **问卷评估**：用户填写关于生活习惯、护肤诉求、现有护肤流程的问卷。
  - **手动输入**：允许用户手动补充或修正皮肤信息。
- **FR1.3 档案动态更新**：每次新的检测结果、日记记录都会更新到档案中，形成历史对比。

### 3.2 模块二：产品与知识库

- **FR2.1 产品数字护照**：为每个谷雨产品建立详情页，包含官方介绍、成分列表、功效解析、适用肤质、使用方法视频。
- **FR2.2 成分词典**：点击任一成分可查看其科学解释、主要功效、安全性及在谷雨产品中的应用故事。
- **FR2.3 产品搭配推荐**：根据用户皮肤档案，智能推荐“精华水+乳液+面霜”等组合，并说明搭配逻辑。

### 3.3 模块三：护肤日记与追踪

- **FR3.1 每日打卡**：提供模板化记录，可快速记录当日使用的产品、皮肤状态（如紧绷、出油）、上妆服帖度等。
- **FR3.2 图文日记**：支持上传皮肤特写照片和文字，记录特殊事件（如过敏、熬夜后）。
- **FR3.3 数据可视化**：以折线图、雷达图等形式展示关键皮肤指标（如含水量、痘痘数量）随时间的变化趋势。

### 3.4 模块四：智能分析与建议引擎

- **FR4.1 个性化护肤方案**：基于皮肤档案，生成包含日/夜流程、产品推荐、使用手法的定制方案。
- **FR4.2 归因分析与洞察**：
  - **正向反馈**：如“连续使用‘光感精华’两周，检测到您的肤色均匀度提升了10%”。
  - **问题预警与建议**：如“近期黑头问题加重，建议您每周增加一次清洁面膜，并检查卸妆是否彻底”。
- **FR4.3 场景化智能提醒**：
  - **季节性**：“明日降温，请注意加强保湿。”
  - **行为性**：“您已记录超过一周未敷面膜，记得给肌肤补充营养哦~”

### 3.5 模块五：体验官专属系统

- **FR5.1 任务中心**：品牌方可发布试用、调研、内容创作等任务，体验官领取并完成。
- **FR5.2 报告提交**：提供结构化模板，方便体验官提交包含前后对比图、使用感受、改进建议的正式报告。
- **FR5.3 积分与激励体系**：完成检测、记录日记、提交报告等行为可获得积分，积分可兑换新品、正装、周边等。
- **FR5.4 体验官内测社群**：一个轻量的讨论区，仅供体验官交流使用，形成高质量的UGC池。

### 3.6 模块六：游戏化与互动

- **FR6.1 养肤之旅**：用户的护肤行为对应为一棵虚拟植物的成长，坚持打卡则植物茁壮，反之凋零。
- **FR6.2 成就徽章**：设立“检测达人”、“成分专家”、“连续打卡21天”等徽章，激励用户探索全部功能。

## 4. 非功能性需求

### 4.1 性能需求

- **响应速度**：核心页面（如首页、检测页）加载时间应小于1.5秒。
- **检测分析**：AI皮肤检测从上传图片到生成报告应在10秒内完成。

### 4.2 安全性与隐私

- **数据安全**：用户皮肤照片、护肤日记等敏感数据需加密存储。
- **隐私协议**：明确告知用户数据用途，并获得授权，绝不私自向第三方泄露。

### 4.3 易用性与用户体验

- **界面设计**：遵循微信小程序设计规范，采用谷雨品牌清新、自然的视觉风格。
- **操作流程**：流程清晰，引导性强，确保新手用户能轻松完成首次检测和档案建立。

### 4.4 可扩展性

- **架构设计**：为未来接入更多检测设备（如肌肤检测仪）、增加直播课等功能预留接口。

---

## 5. 成功指标（KPIs）

- **用户增长**：注册用户数，月活跃用户（MAU）。
- **用户参与度**：人均每周使用次数，皮肤检测功能使用率，护肤日记平均填写率。
- **商业价值**：体验官报告提交数量与质量，通过小程序引导的销售转化率，用户对品牌的好感度NPS（净推荐值）提升。
